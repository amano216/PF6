# 流体アメーバ インタラクティブアート - 再現用プロンプト

## 基本プロンプト

以下のプロンプトをAI（Claude、ChatGPT、Gemini等）に入力することで、同様のアプリケーションを再現できます。

```
「漆黒のキャンバスで戯れる、流体アメーバ」というコンセプトで、インタラクティブなWebアプリケーションを作成してください。

要件：
1. 単一のindex.htmlファイルで完結する実装
2. 背景は完全な漆黒（#000000）
3. 透明感のある本物のアメーバのような見た目
4. p5.jsをCDN経由で使用
5. モバイルフレンドリーなタッチ操作

ビジュアル要件：
- アメーバは多層構造で透明感を持つ
- 細胞質、核、細胞内小器官を表現
- 不規則に動く有機的な形状
- 複数のカラーパレット（オーロラ、炎、海、サイケデリック）

インタラクション要件：
- タップ：新しいアメーバを生成、既存のアメーバは反発
- ドラッグ：アメーバを引き寄せる
- ダブルクリック：アメーバが分裂
- デバイスの傾き：重力として作用
- シェイク：全てリセット

技術要件：
- 波紋エフェクト付きのタッチフィードバック
- 滑らかな60fpsアニメーション
- レスポンシブデザイン
- 控えめなサウンドエフェクト（Howler.js使用）
```

## 詳細版プロンプト（より正確な再現用）

```
本物のアメーバのような透明で有機的な生命体をシミュレートするインタラクティブアートを作成してください。

### 技術スタック
- p5.js（クリエイティブコーディング）
- Howler.js（サウンド）
- 単一HTMLファイル実装
- CDN経由でライブラリを読み込み

### アメーバの構造
1. **外観**
   - RGBAで透明度50程度の適度な透明感
   - 3層構造で内側ほど濃くなるグラデーション
   - 白い細胞膜のハイライト（opacity: 30）

2. **内部構造**
   - 動く核（本体の30%サイズ）
   - 白い核小体
   - 3-8個のランダムな細胞内小器官
   - 全ての内部要素が独立して漂う

3. **形状**
   - 20頂点のcurveVertexで滑らかな輪郭
   - 各頂点が独立したsin波で変形
   - ベースサイズ40-80px

### 物理演算
- 速度ベースの移動（vx, vy）
- 0.95の減衰係数
- 画面端で跳ね返り（速度*-0.5）
- 重力センサー対応

### インタラクション詳細
1. **タップ反応**
   - タッチ位置と逆方向に反発（速度+5）
   - 接触部分の形状が凹む（radius * 0.7）
   - 白い波紋エフェクト生成

2. **分裂メカニズム**
   - ダブルクリックで発動
   - 元のサイズの70%の2体に分裂
   - 最小サイズ40px以下は分裂不可

3. **最大10体まで生成可能**

### カラーパレット（RGBA形式）
- aurora: [[100,255,200,50], [100,200,255,50], [255,100,200,50], [200,100,255,50]]
- fire: [[255,100,50,50], [255,200,50,50], [255,255,100,50], [255,150,0,50]]
- ocean: [[50,100,200,50], [50,150,250,50], [100,200,255,50], [150,255,255,50]]
- psychedelic: [[255,100,255,50], [100,255,255,50], [255,255,100,50], [255,100,200,50]]

### パフォーマンス最適化
- requestAnimationFrameで60fps維持
- 背景は毎フレーム黒で塗りつぶし（尾を引かない）
- タッチイベントでreturn false（デフォルト動作防止）
```

## 段階的実装プロンプト

より細かい制御が必要な場合は、以下の順序でプロンプトを使用：

### ステップ1：基本構造
```
p5.jsを使用して、黒い背景に透明なアメーバを描画する基本的なHTMLファイルを作成してください。
アメーバはcurveVertexを使用した有機的な形状にしてください。
```

### ステップ2：内部構造の追加
```
アメーバに以下の内部構造を追加してください：
- 中心からずれた位置にある核
- 核の中の核小体
- ランダムに配置された細胞内小器官（3-8個）
全て極めて透明（opacity 10-40）にしてください。
```

### ステップ3：インタラクションの実装
```
以下のインタラクションを追加してください：
- マウス/タッチでアメーバが反発する動き
- ドラッグでアメーバを引き寄せる
- ダブルクリックで分裂（70%サイズの2体に）
- タッチ時に白い波紋エフェクト
```

### ステップ4：仕上げ
```
以下の機能を追加して完成させてください：
- 4種類のカラーパレット切り替えUI（右上）
- デバイスの傾きで重力効果
- シェイクでリセット
- Howler.jsで控えめなサウンドエフェクト
```

## トラブルシューティング用プロンプト

問題が発生した場合：

```
アメーバが透明すぎる場合：
「アメーバの透明度を少し上げて、RGBAのアルファ値を50-70の範囲に調整してください」

動きが不自然な場合：
「物理演算を改善して、速度ベースの移動（vx,vy）と0.95の減衰係数を使用してください」

形状が硬い場合：
「curveVertexを使用し、各頂点をsin波で独立してアニメーションさせてください」
```
